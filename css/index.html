    CSS Basics

    CSS stand for cascading style sheets. It is basically a set of rule and follow this pattern: selector { property: value; } . There are so many properties. Do not panic! You can find the information online and you do not need to remember them all.

    How to Include Styles

    Inline styles are not best practice. The best way to include styles is in a separate stylesheet. Use the link element to add the stylesheet. This is best as now our styles can be accessed by across our program.

    Colors

    Named colors: Most browser recognise around 140 colors. These are the most limited! Most monitors can display 16,000,000

    R,G,B Colors

    R,G,B is an additive color system. There are three channels ranging from 0 - 255. Here are some examples: rgb(255,0,0) rgb(0,255,0) rgb(0,0 255)

    Hexidecimal

    Hexidecimal is base 16, we use numbers 0-9 and letters A-F. Basically F is the same as writing 15, this is so we can store more value in a single digit. Just like RGB the first two are red, second two are green and the last two are blue!Hex example: #ff23f2

    Pixels,Font Size and Font Family

    We have many units to specify font size. Pixels are the most commonly used unit in absolute units. Not recommnded for responsive sites! font-size: 100px

    Fonts are built into your browser. We can use custom fonts too.font family: verdana.

    CSS Selectors

    Universal selectors select everything, although it is not that useful in more complex projects. * {background-color: cyan}. The elements selector are more common and can be used to select more than one element when separated by a comma.h1, h2 {color: magenta}

    Id and Class Selector

    We can use the class and ID selector to style elements with a certain ID or class. The id is unique and there is only one per page used sparingly, there can be multiple elements with the same class.

    Desendant Selectors

    This will select all anchor tags nested inside the li li a {color: teal}.. For the example below we could add a class to the ul then style the li and the nested links like this: .descendant li a { text-decoration: none }.

    Adjacent and Descendant Selectors

    This will select elements that are on the same level. So for example a button directly after an input will be pink:
    input + button {background-color: pink}.

    Any element that is a direct descendant can be selected using the angle bracket although it is not so common.
    div > li {color: white}

    Attribute Selector

    We can use square brackets to grab a selector by the type for example!
    input[type="text"] { width: 300px; color: yellow;}

    Psuedo Classes

    Psuedo classes are modifiers used to show when an element has a different interaction such as hover. Psuedo classes start with a colon. To add a psuedo class to the button below I used this selector.
    .hover:hover { background-color: #37bad6; color: #2b2954; }.

    We can also add an active selector for when we click on a button.
    .hover:active { background-color: pink; color: #2b2954; }

    Nth of type can be used to apply styles to every other one, or the last one.
    .box:nth-of-type(2n) { background-color: pink;}.

    Psuedo Elements

    This keyword lets you style a particular part of an element. For example the bullets in a list...

    • ::after
    • ::before
    • ::first-letter
    • ::first-line
    • ::selection

    A final note...The order you declare your styles matters. If you try to add a color selector to a h1 for example twice the last selector will win.

    WTF is Specificity?

    What happens when there is a conflict in styles? What if we have a button with a class and just a regular button? The browser measures how specific a selector is and the most specific will win. This is the general formula to remember:
    ID > CLASS > ELEMENT. Order also matters, so the last style selector in the sheet will have precidence. Use dev tools to explore when styles are not applied in the way you expected.

    Inline Styles

    In line styles are the most specific way of styling elements. but they are not commonly used. Avoid these as they are confusing! If you use the important! exception this will overide all styles.

    CSS Inheritance

    This is something we have seen many times. Basically child elements will inherit styles from parent elements. So if the body is set as a color of purple, all elements in the body will be purple. We can inspect this in the dev tools. We can use the keyword inherit to make sure elements inherit as sometimes elements do not inherit.

    Margin and Padding

    Padding

    Padding pads the space between the content and the border. We can apply padding to each side separately or we can use the shorthand property to add padding to all four sides to any element.padding: 20px
    . We can also set padding to all four sides using the order top | right | bottom | left. If we inspect an element usign dev tools we can see padding as a green highlight around an element.

    Margin

    Margin is the space outside the element's border. Just like padding we can add to indivisual sides or all four sides using the shorthand property. By default some elements have margin. Like the body. It is common to set body to
    body { margin: 0}

    Display Property

On our heading elements we could change the properties display: inline. What happens to padding and margin on inline elements? Padding does not take space on inline elements and is not respected by other elements on the page. Width, height, margin and padding are all repsected on block elements.

    CSS units

    Units in CSS consist of relative and absolute. Pixels are the most common absolute value. We will focus on the relative units that consist of...

    • EM
    • REM
    • VH
    • VW
    • %

    Percentages are always relative to some other value. For example width:50%. This describes an element that is 50% of the parent element.

    Em's and Rem's are relative units. With font size the value is based on the parent element for example...If a parent has a font size of 30px and the child has font size of 2em that will be 60px. If we use em's with the margin property for example a h2 with a font size of 2em and we add a margin of 1em, margin-left: 1em is equal to 2em. This is useful because it allows us to make elements responsive. With pixels elements do not scale well. When we use ems element proportions will grow or shrink relative to the parent.

    The problem with ems is they can grow and shrink quickly. As they can multiply if we have nested elements

    Alpha Channel

    This governs the transparancy of a color. This rgba(255, 255, 255, 0) will result in full transparency. Whereas this rgba(255, 255, 255, 1) will result in no transparency. If we adapt the alpha channel to .5 this will be opaque. it is worth noting that this only effects the background.

    Opacity applies to the entire element.Other elements will also be impacted such as any buttons. Example:opacity: 50%

    Position

    This can be super confusing! You may need to refer to the docs for this one! Let's start with three divs.

    Static

    The default position of any element is static

    Relative

    We can use position relative to offset an element relative to itself.position: relative; top: 100px;

    Absolute

    We can use position absolute to remove the element from the normal flow of the document and no space is held for that element.position: absolute; top: 100px;. If the parent does not have a position of absolute, the element will be positioned relative to the body.

    Fixed

    An element that is fixed will go all the way up to the containing block and stay there. Even when there is additional content and the user scrolls. Sticky is simlilar it will remain inplace until the user scrolls past and then it will "stick" to the top of the window.

    Flexbox

    An introduction to Flexbox. Flexbox allows us to layout content dynamically. There are two axis to consider when using flexbox. The main axis that flows from left to right. We can use row-reverse to change the direct to right to left. We can do teh same thing with the column property which is the cross axis and flows top to bottom column-reverse.

    Justify-content is useful for organising our content. We can use the following properties...

    • justify-content:start
    • justify-content:flex-end
    • justify-content:center
    • justify-content:space-between
    • justify-content:space-around

    Elements will automatically fit to a containers content. We can use flex-wrap: wrap to ensure items go on a new row when they do not fit.

    Align items is used to align items on the cross axis. The default cross axis is top to bottom. We can use the following properties with align-items... All are self explanatory apart from baseline. Baseline aligns each element to the bottom of the text.

    • align-items: flex-start
    • align-items: flex-end
    • align-items: center
    • align-items: base-line

    Align content is used to control space between colums and it only works when we have wrap or wrap-reverse as a property that is works. Aloign self is very similar and we add it to indivual elements rather that the flex container it self.

    How do we make elements grow or shrink based on the flex container? We can use flex-basis. Flex basis is the initial size an element should be wheh added to our content. Flex-grow ensures an element takes up all the space available. Flex-shrink ensures an element reduces in size relative to other elements.
